# AGENTS Guide

## 使用本仓库的通用原则

- 保持通用性：不要写入与单一公司项目强耦合的路径、命令或私有基础设施信息。
- 最小可维护：优先补充结构化文档和模板，再逐步增加实现。
- 配置分离：仓库仅保存 `*.example` 示例，真实配置文件必须忽略。
- 可审计：所有新增 Agent/MCP 都应有明确输入输出约定和安全边界。

## 如何新增一个 Agent（模板流程）

1. 复制 `agents/templates/agent.md` 作为新 Agent 文档。
2. 填写名称、目标、输入输出约定、可用工具、禁区、示例对话。
3. 将文档放入合适目录（例如 `agents/company-dev/`）。
4. 在 `README.md` 或对应索引文档中补充登记（TODO: 可后续增加索引机制）。

## 如何新增一个 MCP 工具（模板流程）

1. 在 `mcp/` 下创建工具目录。
2. 参考 `mcp/templates/README.md` 补齐用途、接口、运行方式、配置项、测试方式。
3. 在 `configs/` 下补充对应 `*.example` 配置模板。
4. 明确工具访问边界和脱敏规则，并同步到安全文档（如适用）。

## 敏感信息与安全边界

- 严禁提交任何敏感信息（真实内网信息、密钥、令牌、Cookie、账号等）。
- 禁止将个人信息、客户数据、内部机密内容写入模型上下文或模板示例。
- 详细规范见：`docs/SECURITY.md`
